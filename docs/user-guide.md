# 律所操作手册

本手册面向律所业务人员（律师、律师助理），无需技术背景即可操作。

---

## 什么是 AI 模拟法庭？

这套系统会模拟一场完整的民事庭审。4个 AI 角色各司其职：

| 角色 | 做什么 | 类比 |
|------|--------|------|
| 📋 书记员 | 宣布开庭、控制流程 | 真实法庭的书记员 |
| 🔴 原告律师 | 宣读起诉状、举证、辩论（极强攻击性） | 你的对手律师（最厉害版本） |
| 🔵 被告律师 | 答辩、质证、反驳（极强防御性） | 你的对手律师（最厉害版本） |
| ⚖️ 法官 | 归纳焦点、判决、出报告 | 经验丰富的法官 |

**核心价值**：在正式开庭前，用 AI 模拟出最激烈的对抗场景，提前发现你的证据链漏洞、预判对方可能的策略、评估胜诉概率。

## 准备案件材料

### 你需要准备以下文件（Markdown 或纯文本格式）

创建一个案件文件夹，按以下结构组织：

```
我的案件/
├── case-brief.md              # 案件概要（必需）
├── complaint.md               # 起诉状（必需）
├── defense.md                 # 答辩状（必需）
├── plaintiff-evidence/        # 原告证据文件夹
│   ├── evidence-001.md        # 证据1
│   ├── evidence-002.md        # 证据2
│   └── ...
├── defendant-evidence/        # 被告证据文件夹
│   ├── evidence-001.md
│   └── ...
├── plaintiff-strategy.md      # 原告诉讼策略（可选，AI不会泄露给对方）
└── defendant-strategy.md      # 被告抗辩策略（可选）
```

### case-brief.md 怎么写？

```markdown
# 案件概要

## 基本信息
- 案由：XX纠纷
- 原告：张某
- 被告：李某
- 标的额：XX元

## 案件简述
（用3-5段话概述案件经过，包括：起因、关键时间节点、争议焦点）
```

### 证据文件怎么写？

每份证据一个文件：

```markdown
# 证据X：借款合同

## 证据信息
- 证据名称：《借款合同》原件
- 证据类型：书证
- 证明目的：证明双方存在借贷关系

## 证据内容摘要
（合同的关键条款摘要，不需要全文，但关键条款要准确）
```

### 关于"策略文件"

`plaintiff-strategy.md` 和 `defendant-strategy.md` 是**私密文件**，AI 原告/被告会参考但不会在庭审中暴露给对方。你可以在里面写：

- 己方的弱点和应对方案
- 希望 AI 重点攻击的方向
- 不希望暴露的信息
- 和解底线

## 操作步骤

### 第一步：初始化案件

请技术人员或你自己在终端中运行：

```bash
cd moot-court-ai
./scripts/init-case.sh 我的案件/
```

你会看到：
```
✅ 起诉状
✅ 原告证据 3 份
✅ 答辩状
✅ 被告证据 2 份
✅ 案件概要 -> 书记员 + 法官
✅ 案件初始化完成
```

### 第二步：启动系统

```bash
openclaw gateway --port 18789
```

### 第三步：打开浏览器

访问 http://localhost:18789

首次访问需要输入 token（技术人员会告诉你）。

### 第四步：触发庭审

在 WebChat 对话框中输入：

> 请启动模拟法庭，案件已准备就绪。

系统会自动开始庭审流程。你会看到 4 个 AI 依次发言：

1. 书记员宣布开庭
2. 原告陈述
3. 被告答辩
4. 举证质证（可能多轮）
5. 法官归纳争议焦点（**此处会暂停，需要你确认**）
6. 双方辩论
7. 最后陈述
8. 法官宣判（**此处会暂停，需要你确认**）

### 第五步：查看结果

庭审结束后，法官会输出两份文档：

1. **《模拟民事判决书》** — 完整的判决书，包含事实认定、法律适用、判决主文
2. **《诉讼风险推演报告》** — 包含：
   - 原告胜诉概率
   - 证据强弱评估
   - 对方可能的上诉理由
   - 和解建议区间

## 使用技巧

### 1. 模拟"最坏情况"
在被告策略文件里写上你能想到的最强抗辩理由，让 AI 被告全力施压。如果你的证据链在最强对手面前依然站得住，真实庭审就更有把握。

### 2. 反复模拟
修改策略文件后重新初始化并运行，可以测试不同策略的效果。

### 3. 关注"争议焦点"
法官归纳的争议焦点是整场庭审的灵魂。如果你发现 AI 法官归纳的焦点里有你没预料到的，说明你的庭前准备可能存在盲区。

### 4. 关注被告的"三性质证"
AI 被告会对你的每份证据进行真实性、合法性、关联性的极限质证。如果某份证据被成功挑战，说明该证据在真实庭审中可能也有问题，需要补强。

### 5. 利用风险报告做和解谈判
风险报告中的"和解建议区间"可以作为与对方谈判的参考基准。

## 注意事项

- **AI 不是法律意见**：模拟结果仅供参考，不构成正式法律意见。最终判断仍需资深律师把关。
- **数据安全**：案件材料会通过 API 发送给模型提供商（DeepSeek / 阿里云）进行处理。如涉及高度敏感信息，请咨询技术人员是否启用本地模型。
- **成本**：每次完整庭审模拟约 ¥2-5，由 API 调用产生。
- **结果可能不同**：同一案件多次模拟，结果可能略有差异（AI 的随机性），这是正常的。多次模拟取共识结论更可靠。
